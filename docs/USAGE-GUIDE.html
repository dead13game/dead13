Session 命名规范

claude -n "类型-层-简述"

类型: feature / fix / perf / refactor / explore / content
层:   game / pixi / anim / ui / multi
简述: 中文，3-8 字
我要做什么 命令示例
加新角色/技能 claude -n "feature-game-丽莎"
改战斗公式 claude -n "feature-game-伤害调整"
修牌桌布局 bug claude -n "fix-pixi-竖屏重叠"
改按钮样式 claude -n "fix-ui-按钮过小"
修动画不动 claude -n "fix-anim-飞牌卡住"
数值平衡 claude -n "content-角色胜率"
性能优化 claude -n "perf-移动端帧率"
跨层重构 claude -n "refactor-multi-状态管理"
不知道改什么文件 claude -n "explore-防御逻辑"

提问规范

✅ 好的提问

「4人竖屏，第3回合温迪攻击雷电，控制台 [game] 日志 damage_calc 显示 afterMinus2=8 但 targetHpAfter 只少了 3。预期减少 8 但实际减少 3。完整日志：[粘贴]」
「新增角色丽莎，技能：选择一名玩家，偷取其一张防御牌并造成 5 点伤害。技能限用 2 次，endTurn=false。」

❌ 坏的提问

「防御有 bug」「加个新角色」「帮我优化一下」

提问四要素

  1. 复现步骤 — 几人、什么模式、谁做了什么操作
  2. 日志/报错 — [game] 或 [pixi] 日志输出
  3. 预期行为 — 你觉得应该发生什么
  4. 实际行为 — 实际发生了什么

不知道涉及哪些文件怎么办

不用记。 直接在对话里问 Claude，它会自动 spawn Explore subagent 搜索，或者开 claude -n "explore-问题"。

改代码时用什么 Agent

改的目录 对话里说什么 自动用
src/game/ 「用 game-logic 改 combat.js」 game-logic
src/pixi/ 「用 pixi-render 改 TableLayout」 pixi-render
src/bridge/ 「用 animation 改动画」 animation
src/components/ 「用 vue-ui 改 ActionBar」 vue-ui
跨多个层 主对话自己改 —
debug 时不分派，改代码时再分派。bug 定位需要跨层视角。

Debug 流程

  1. 对话里说 /debug 激活六步调试法
  2. Playwright 打开游戏复现 → 读 browser_console_messages
  3. 筛选 [game] 日志定位事件
  4. 查到根因文件 → module-map.md 确认
  5. 先写测试复现 → 改代码 → npm run test

Session 生命周期

开 Session → 说一句任务描述 → 探索/修改/验证 → 改完即关
一个 session 只做一件事。改一个文件是最佳状态。DeepSeek 上下文超 60% 后质量下降。

跨 Session 信息共享

改接口后更新 CLAUDE.md「Agent 间合约」表格。三个通道：

  1. SessionStart hook — 自动显示上次提交改了哪些文件
  2. Memory 系统 — 变更日志 + 模块映射 + 合约
  3. CLAUDE.md 合约表 — 函数签名、state 字段、桥接接口
